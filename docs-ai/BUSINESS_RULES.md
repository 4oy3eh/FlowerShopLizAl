# BUSINESS_RULES.md — Бизнес-правила FlowerShop

## КРИТИЧЕСКИЕ ПРАВИЛА (нарушение = потеря денег/клиентов)

### R1: Фиксация цен при создании заказа
**Правило:** Цена КОПИРУЕТСЯ из текущего прайса в `order_items.unit_price` и `orders.wrapping_price`/`note_price`/`delivery_price` в момент создания заказа.
**Почему:** Динамическое ценообразование не должно менять стоимость уже оформленных заказов.
**Реализация:** `price_service.snapshot_prices()` вызывается ТОЛЬКО при создании.

### R2: Контроль остатков — жёсткий
**Правило:** Нельзя создать заказ, если `stock_available < requested_quantity` для ЛЮБОГО сорта в заказе.
**Почему:** Невозможно продать то, чего нет.
**Реализация:** `stock_service.check_availability()` ПЕРЕД созданием заказа. Транзакция.

### R3: Предоплата не возвращается
**Правило:** При отмене заказа предоплата не возвращается. Никогда. Без исключений.
**Почему:** Бизнес-решение владельца. Цветы скоропортящиеся.
**Реализация:** `cancel_order()` не создаёт возвратный платёж. Поле `paid_amount` не обнуляется.

### R4: Каждый платёж логируется отдельно
**Правило:** Любое поступление денег = новая запись в `payment_log`.
**Почему:** Жёсткий контроль "кто, когда, сколько, каким способом".
**Реализация:** Никогда не UPDATE `paid_amount` напрямую — только через `payment_service.record_payment()` который создаёт лог И обновляет сумму.

### R5: Отмена возвращает цветы на склад
**Правило:** При отмене заказа все позиции из `order_items` возвращаются в `stock_available`.
**Почему:** Отменённые цветы можно продать другому клиенту.
**Реализация:** `stock_service.release(order_id)` вызывается в `cancel_order()`.

### R6: Нельзя отменить заказ в статусе 'delivering' и далее
**Правило:** Отмена доступна только для статусов: new, confirmed, assembling, ready.
**Почему:** Курьер уже в пути — поздно отменять.
**Реализация:** Проверка `order_status` перед отменой.

## ПРАВИЛА ВАЛИДАЦИИ

### V1: Формат телефона
- Украинский: `+380XXXXXXXXX` (12 символов с +)
- При вводе автоматически форматировать: убрать пробелы, добавить +380 если начинается с 0

### V2: Обязательные поля заказа
- `customer.phone` — всегда
- `customer.name` — "Аноним" если не указано
- `recipient_name` — всегда
- `recipient_phone` — всегда
- `ribbon_color_id` — всегда (лента обязательна)
- Минимум 1 позиция в `order_items`

### V3: Доставка vs Самовывоз
- Если `is_pickup = 0` → `delivery_address` обязателен, `delivery_price` берётся из настроек
- Если `is_pickup = 1` → `delivery_address` NULL, `delivery_price` = 0

### V4: Записка
- Если `has_note = 1` → `note_text` обязателен (не пустой), `note_price` из настроек
- Если `has_note = 0` → `note_text` NULL, `note_price` = 0

### V5: Количество цветов
- Минимум 1 штука любого сорта
- Нет максимума, но проверка остатков (R2)

## ПРАВИЛА ОПЛАТЫ

### P1: Статусы оплаты
```
paid_amount == 0                → 'unpaid'
0 < paid_amount < total_price   → 'partial'
paid_amount == total_price       → 'paid'
paid_amount > total_price        → 'overpaid'
```

### P2: Рекомендуемая предоплата
- Система ПОКАЗЫВАЕТ рекомендуемую предоплату (50% от total)
- Но НЕ БЛОКИРУЕТ создание заказа без предоплаты
- Оператор решает сам

### P3: Переплата
- Если клиент заплатил больше → `overpayment = paid_amount - total_price`
- Переплата отображается, но не обрабатывается автоматически

### P4: Доплата для курьера
- Курьер видит: `remaining = total_price - paid_amount`
- Если `remaining > 0` → крупная надпись "ДОПЛАТА: XXX грн ⚠️"
- Если `remaining <= 0` → надпись "ОПЛАЧЕНО ✓"

## ПРАВИЛА МАРШРУТОВ

### M1: Формирование маршрута
- Только заказы со статусом `ready` + `is_pickup = 0` + `route_id IS NULL`
- Группировка по `desired_time` (временные слоты)
- Ограничение по вместимости авто (из `system_settings.max_bouquets_per_route`)

### M2: Временные слоты
```
08:00-10:00
10:00-12:00
12:00-14:00
14:00-16:00
16:00-18:00
18:00-20:00
20:00-22:00
```
Источник: `config.TIME_SLOTS` — единая точка правды для всех модулей.

### M3: Google Maps URL
- Маршрут: База → Адрес 1 → Адрес 2 → ... → База
- Генерируется автоматически при формировании маршрута
- Курьер открывает ссылку на телефоне

## ПРАВИЛА СКЛАДА

### S1: Расчёт доступности
```
stock_available = stock_total - SUM(order_items.quantity
    WHERE orders.order_status NOT IN ('cancelled', 'done'))
```

### S2: Пересчёт при изменениях
Пересчитывать `stock_available` при:
- Создании заказа (уменьшить)
- Отмене заказа (увеличить)
- Ручной корректировке `stock_total` (пересчитать)

### S3: Индикация остатков
```
stock_available / stock_total:
  > 50%  → зелёный
  20-50% → жёлтый
  < 20%  → красный
  == 0   → серый, "Нет в наличии"
```

## ПРАВИЛА ЦЕНООБРАЗОВАНИЯ

### PR1: Массовое изменение цен
- Админ указывает процент (+10%, -15%)
- Применяется ко ВСЕМ активным сортам
- Каждое изменение записывается в `price_change_log`
- Существующие заказы НЕ затрагиваются

### PR2: Поштучное изменение цены
- Админ меняет цену конкретного сорта
- Записывается в `price_change_log`
- Существующие заказы НЕ затрагиваются
